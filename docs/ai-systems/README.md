# AI統合システム

## 🎯 概要

このディレクトリには、プロジェクトで使用するAI統合システムに関するドキュメントが含まれています。

AI統合システムは以下の3つの主要コンポーネントで構成されています:

1. **ガードシステム** - AIの作業をリアルタイムで監視・保護
2. **失敗学習システム** - 過去の失敗から学習し、同じミスを防ぐ
3. **メンテナンスシステム** - プロジェクト品質の自動監視・維持

## 🚀 使い方

### 1. AI機能を有効化

開発者ツール（F12）のコンソールで実行:

```javascript
localStorage.setItem('enable-ai-coordination', 'true');
location.reload();
```

詳細: [how-to-enable.md](./how-to-enable.md)

### 2. ガードシステムの起動

```bash
# 統合ワークフロー（推奨）
npm run ai:guard

# または個別実行
npm run ai:check-failures
npm run ai:detect-danger
```

詳細: [guard-system.md](./guard-system.md)

### 3. 失敗学習の実行

```bash
# Git履歴から学習
npm run ai:learn-from-git

# 失敗履歴から学習
npm run ai:learn-failures
```

詳細: [failure-collection.md](./failure-collection.md)

## 🛡️ システムの動作フロー

```
Phase 0: 学習期間
├─ Git履歴から過去の失敗を全学習
└─ 十分なデータが蓄積されるまで待機

Phase 1: リアルタイムガード
├─ AI修正開始を検知
├─ 過去の類似失敗を検索
├─ 危険パターンを検出
└─ 警告・チェックリストを表示

Phase 2: 失敗収集
├─ 修正失敗を記録
├─ 失敗パターンを分析
└─ 水先案内人データを更新

Phase 3: 継続学習
├─ 新しい失敗パターンを学習
├─ ガードルールを更新
└─ 次回の修正に活用
```

## 📊 システム構成

```
AI統合システム
│
├─ ガード層（事前防止）
│  ├─ リアルタイムガードシステム
│  ├─ 警告システム
│  └─ チェックリスト表示
│
├─ 学習層（継続改善）
│  ├─ Git履歴学習
│  ├─ 失敗パターン分析
│  └─ 水先案内人データ生成
│
└─ メンテナンス層（品質維持）
   ├─ 品質神経系統
   ├─ 自動監視
   └─ 統合パイプライン
```

---

## 🎓 学習データ

システムは以下のデータソースから学習します:

1. **Git履歴** - コミットメッセージ、差分、ファイル変更
2. **失敗履歴** - `.aitk/ai-failure-history.json`
3. **品質メトリクス** - テストカバレッジ、ビルド成功率
4. **ユーザーフィードバック** - 手動で追加された失敗パターン

## ⚠️ 重要な設計思想

```
❌ 学習不十分な段階での通知 = ノイズ（誤報）
✅ 十分な学習後の警告 = 効果的な失敗防止
```

システムは十分なデータが蓄積されるまで警告を抑制し、学習が完了してから本格的に動作します。
