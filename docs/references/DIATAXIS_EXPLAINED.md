# Diátaxisって何？ - 小学生でもわかる説明

**対象読者**: ドキュメント整理に興味がある全ての人  
**所要時間**: 5分  
**前提知識**: なし

---

## 🤔 一言で言うと

**「本屋さんの棚分け」と同じ考え方です。**

本屋さんに行くと、こんな風に分かれていますよね：

```
📚 児童書コーナー     → 物語を楽しむ
🔧 実用書コーナー     → 問題を解決する
📖 読み物コーナー     → 知識を深める
📋 辞書・事典コーナー → 情報を調べる
```

Diátaxis（ディアタクシス）は、**ドキュメントを4つの棚に分ける考え方**です。

---

## 📊 4つの棚

### 1. 🎓 Tutorials（チュートリアル）= 「料理教室」

**特徴**: 先生と一緒に、順番通りに作る

```
例：カレーの作り方教室
─────────────────
① 野菜を切ります
② お肉を炒めます
③ 水を入れます
④ ルーを溶かします

完成！ 🍛
```

**このプロジェクトでの例**:
- 「初めてのメタAI学習アプリ」
- 「文法問題の作り方（1から順番に）」

**いつ使う？**
- 初めてこのプロジェクトに触る人
- 「とりあえず動かしてみたい」とき

---

### 2. 🔧 How-to Guides（ハウツーガイド）= 「工具箱」

**特徴**: 問題が起きたら、必要な道具を取り出す

```
例：家の修理
─────────────────
問題：水漏れ → 🔧 パッキン交換ガイド
問題：電球切れ → 💡 電球の取り替え方
問題：鍵が固い → 🔑 鍵穴の油差し方
```

**このプロジェクトでの例**:
- [メタAIトラブルシューティング](../guidelines/META_AI_TROUBLESHOOTING.md)
- [デプロイメント運用ガイド](../processes/DEPLOYMENT_OPERATIONS.md)
- [リファクタリング安全ガイド](../processes/REFACTORING_SAFETY.md)

**いつ使う？**
- 「復習単語が出題されない！」（緊急）
- 「本番環境にデプロイしたい」（特定タスク）
- 「コードを安全にリファクタリングしたい」

---

### 3. 📖 Explanation（説明）= 「読み物」

**特徴**: 「なぜそうなっているのか」を理解する

```
例：料理の科学
─────────────────
Q: なぜ肉を焼くと美味しくなるの？
A: メイラード反応で香ばしい成分が…

Q: なぜパンは膨らむの？
A: イーストが二酸化炭素を…
```

**このプロジェクトでの例**:
- [適応的学習ネットワークアーキテクチャ](../design/ADAPTIVE_NETWORK_ARCHITECTURE.md)
- [学習フェーズアーキテクチャ](../design/LEARNING_PHASE_ARCHITECTURE.md)
- [DTAシミュレーション設計](../design/DTA_SIMULATION_DESIGN.md)

**いつ使う？**
- 「なぜこの設計になっているの？」
- 「背景を知りたい」
- 「コンセプトを理解したい」

---

### 4. 📋 Reference（リファレンス）= 「辞書」

**特徴**: 正確な情報をさっと調べる

```
例：料理の辞書
─────────────────
「小さじ1」→ 5ml
「中火」  → 150-170℃
「弱火」  → 120-150℃
```

**このプロジェクトでの例**:
- [QuestionScheduler仕様書](../specifications/QUESTION_SCHEDULER_SPEC.md)
- [NEW HORIZON公式単元構成](NEW_HORIZON_OFFICIAL_UNIT_STRUCTURE.md)
- [API仕様書（全26ファイル）](../specifications/)

**いつ使う？**
- 「この関数の引数は？」
- 「Grade 2のUnitは何個？」
- 「正確なデータ構造を確認したい」

---

## 🎯 なぜ4つに分けるの？

### ❌ 分けないと困ること

**全部ごちゃ混ぜの例**:
```
📄 「メタAIについて.md」
─────────────────
・メタAIとは何か（説明）
・メタAIの使い方（チュートリアル）
・問題が起きたときの直し方（ハウツー）
・API仕様（リファレンス）
・設計思想（説明）
・設定方法（ハウツー）

👤「え、今すぐ問題を直したいだけなのに、
   どこを読めばいいの？」
```

### ✅ 分けるとこうなる

```
🎓 [初めてのメタAI](tutorial)      ← 初心者はここから
🔧 [トラブルシューティング](how-to) ← 問題が起きたらここ
📖 [設計思想](explanation)          ← 理解を深めたいときはここ
📋 [API仕様](reference)             ← 正確な情報はここ
```

**探しやすい！読みやすい！**

## 📚 実際の本屋さんで例えると

### ケース1: カレーを初めて作る

```
👤「カレー作ったことないけど作りたい」

本屋で買う本：
🎓 「はじめてのカレー（写真付き手順書）」← Tutorial
```

### ケース2: カレーが辛すぎた

```
👤「カレーが辛すぎる！どうしよう！」

本屋で探す本：
🔧 「料理の困ったを解決！ Q&A100」← How-to
   → Q47: カレーが辛すぎたら？
```

### ケース3: もっと美味しくしたい

```
👤「なぜプロのカレーは美味しいんだろう」

本屋で読む本：
📖 「カレーの科学」← Explanation
```

### ケース4: スパイスの分量を調べる

```
👤「ガラムマサラは何グラム？」

本屋で調べる本：
📋 「スパイス辞典」← Reference
```

## 🎨 図解: 4つの違い

```
              学習目的              タスク目的
              ↓                    ↓
        ┌──────────┐      ┌──────────┐
        │            │      │            │
実践的  │ 🎓Tutorial │      │ 🔧 How-to  │
        │            │      │            │
        │順番通りに  │      │問題を     │
        │手を動かす  │      │解決する   │
        └──────────┘      └──────────┘
              ↑                    ↑
              学ぶ                 使う
              ↓                    ↓
        ┌──────────┐      ┌──────────┐
        │            │      │            │
理論的  │📖Explanation│      │📋Reference │
        │            │      │            │
        │背景を     │      │正確な情報を│
        │理解する   │      │調べる     │
        └──────────┘      └──────────┘
```

## 🤝 中年の方へのアドバイス

### 思い出してください

**昔、ワープロやパソコンを覚えたとき**:

1. まず「基礎講座」に通った → 🎓 Tutorial
2. 分からないことは「Q&A本」で調べた → 🔧 How-to
3. 雑誌で「なぜWindowsは…」という記事を読んだ → 📖 Explanation
4. 操作方法は「リファレンスマニュアル」を見た → 📋 Reference

**同じことです！**

## ✅ まとめ: 3行で

1. **Diátaxis = ドキュメントを4つに分ける考え方**
2. **Tutorial/How-to/Explanation/Reference = 料理教室/工具箱/読み物/辞書**
3. **目的に応じて読む場所が変わる → 探しやすい！**

## 💡 おまけ: 分類に迷ったら

```
このドキュメントは…

✅ 初心者が順番通りに進むもの？
   → 🎓 Tutorial

✅ 特定の問題を解決するもの？
   → 🔧 How-to

✅ 背景や理由を説明するもの？
   → 📖 Explanation

✅ 正確な情報を記載しているもの？
   → 📋 Reference
```

**複数に該当する？**
