# WORKING SPEC

このファイルは「強制力が働く仕様（作業の正）」です。

## 0. 原則（守れない場合は必ず質問）

- 不明点や矛盾がある場合は、推測で実装しない。まず質問して確定する。
- 回帰を出さない。仕様はテスト/ガード/チェックで固定する。
- 仕様変更・運用決定は必ず決定ログに残す（DECISIONS.md）。

## 1. UI実装の強制ルール（今回追加）

UI・レイアウト・スタイルについて「同じ仕様/レイアウト/スタイルで実装して」と指示がある場合：

- 既存のコンポーネント/スタイル/トークンを**流用**する。
- 独自アレンジ（新しい色・フォント・影・余白ルール・新テーマ等）を勝手に足さない。
- 変更は**最小限**にする（既存を壊さず、必要な差分だけ）。

### 例（やること）
- 既存コンポーネントの再利用（props調整や軽微な配置変更）
- 既存CSS/Tailwindトークンの使用

### 例（やらないこと）
- 新しいデザインシステムの導入
- 新しい色やシャドウをハードコード
- 指示に無いUI拡張（新ページ/新モーダル/余計な演出）
