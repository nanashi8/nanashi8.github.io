# スコアリングアルゴリズム（今後の拡張案）

このドキュメントは「スコアリング周りの将来案」を集約するためのスタブです。

## 背景

- 既存実装は安定性を優先し、過度に複雑化しない方針
- ただし、長期運用では「評価軸の追加」「説明可能性」「閾値調整」などが必要になり得る

## 目標（案）

- スコアの意味を説明できる（ログ/根拠/寄与度）
- 誤検知（不要な警告/不要なブロック）を減らす
- 変更の影響を測定しながら安全に調整できる
