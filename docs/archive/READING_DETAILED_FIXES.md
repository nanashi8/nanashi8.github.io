## 🔍 修正時の注意点

### JSON形式のチェック

修正後は必ずJSON形式が正しいか確認してください:

```bash
# JSONの構文チェック
jq . public/data/advanced-1.json > /dev/null && echo "OK" || echo "Error"
```

### カンマの位置

並列要素を修正する際、読点（、）を適切な位置に挿入してください:

✅ 良い例:
```
"気候変動、資源枯渇、汚染は"
```

❌ 悪い例:
```
"気候変動,資源枯渇,汚染は"  // 半角カンマ
"気候変動 、資源枯渇 、汚染は"  // スペースあり
```

### 文の流れ

日本語として自然に読めるかを確認:

```
元: "これらには栄養運動睡眠精神的健康が含まれます"
→ 読みにくい

修正: "これらには栄養、運動、睡眠、精神的健康が含まれます"
→ 読みやすい
```
