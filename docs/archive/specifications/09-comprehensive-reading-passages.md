## 📌 概要

高校受験対策用の包括的な英語長文読解コンテンツを作成する。
難易度別（初級・中級・上級）に合計11パッセージを作成し、CSVデータベースの語彙を効率的にカバーする。

## 📝 直訳表示ルール

### 関係代名詞

| 語 | 先行詞 | 直訳表示 |
|---|---|---|
| who | 人 | 関係代名詞(その人は) |
| which | 物 | 関係代名詞(その物は) |
| which | 動物 | 関係代名詞(その動物は) |
| that | 人 | 関係代名詞(その人は) |
| that | 物 | 関係代名詞(その物は) |
| that | 動物 | 関係代名詞(その動物は) |

### その他の文法要素

#### 接続詞

```text
that(接続詞) → "〜ということ"
because → "なぜなら"
although → "〜だけれども"
while → "〜する間に / 〜である一方で"
since → "〜なので / 〜以来"
unless → "〜でない限り"
```

#### 不定詞
```
to + 動詞(目的) → "〜するために"
to + 動詞(名詞的用法) → "〜すること"
to + 動詞(形容詞的用法) → "〜するための"
```

#### 助動詞
```
will → "〜だろう / 〜するつもり"
would → "〜だろう(過去) / 〜するつもりだった"
should → "〜すべき"
must → "〜しなければならない / 〜に違いない"
may → "〜かもしれない / 〜してもよい"
#### 助動詞

```text
can → "〜できる"
could → "〜できた / 〜かもしれない"
```

#### 完了形
```
have/has + 過去分詞 → "〜してしまった / 〜したことがある"
had + 過去分詞 → "〜していた(過去完了)"
```

#### 受動態

```text
be + 過去分詞 → "〜される"
be動詞 + 過去分詞 + by 〜 → "〜によって〜される"
```

#### 前置詞（位置関係を明確に）
```
in → "〜の中に"
on → "〜の上に"
at → "〜に(場所・時点)"
by → "〜のそばに / 〜によって"
for → "〜のために / 〜の間"
with → "〜と一緒に / 〜を使って"
```

## 🔍 語彙選定の厳守ルール

### ❌ 絶対に避けるべきこと

1. **初級パッセージに中級・上級語彙を含めない**
   - 初級は `difficulty == "初級"` の語彙のみ
   - 例外: 基本的な機能語（a, the, is など）

1. **中級パッセージに上級語彙を含めない**
   - 中級は `difficulty == "初級" OR "中級"` の語彙のみ
   - 初級語彙の使用は問題なし

### ✅ 推奨される語彙使用

1. **上級パッセージは全難易度を使用可能**
   - 初級・中級語彙を土台に上級語彙を追加
   - 自然な文章構成を優先

1. **各語彙の適切な出現頻度**
   - 重要語彙: 2-3回（文脈で意味を定着）
   - 基本機能語: 制約なし
   - 難易度語彙: 最低1回は出現

## 📐 フレーズ分割(チャンク)ルール

### 基本原則

- 英語ネイティブが自然に話す区切りで分割
- 意味の塊（semantic chunk）を維持
- 文法構造ではなく、リズムと意味で判断

### 難易度別チャンク長

**初級:**
- 平均: 3-5語
- 最大: 7語
- 理由: 理解しやすさを優先

**中級:**
- 平均: 5-7語
- 最大: 10語
- 理由: 自然な英語の流れとバランス

**上級:**
- 平均: 7-10語
- 最大: 15語
- 理由: 複雑な構文を自然な単位で

### チャンク分割の良い例

```typescript
// ❌ 悪い例（文法的すぎる）
["The student", "who won the prize", "was very happy"]

// ✅ 良い例（自然な話し言葉の区切り）
["The student who won the prize", "was very happy"]

// ❌ 悪い例（長すぎて理解困難）
["In order to achieve success in the modern world where technology is constantly evolving"]

// ✅ 良い例（適切な長さで分割）
["In order to achieve success", "in the modern world", "where technology is constantly evolving"]
```

## 📋 データ構造

### ReadingPassage型
```typescript
export interface ReadingPassage {
  id: string;              // "beginner-1", "intermediate-3", "advanced-2"
  title: string;           // パッセージタイトル
  level: "beginner" | "intermediate" | "advanced";
  theme: string;           // テーマ（日本語）
  targetWordCount: number; // 目標語数
  actualWordCount: number; // 実際の語数
  phrases: ReadingPhrase[];
}
```

### ReadingPhrase型
```typescript
export interface ReadingPhrase {
  id: string;              // "phrase-1", "phrase-2"
  words: string[];         // 単語の配列
  phraseMeaning: string;   // フレーズ全体の和訳
  segments: ReadingSegment[];
  isUnknown: boolean;      // 未使用（後方互換性のため保持）
}
```

### ReadingSegment型
```typescript
export interface ReadingSegment {
  word: string;      // 単語（英語）
  meaning: string;   // 単語の意味（直訳ルールに従う）
  isUnknown: boolean; // 分からない単語フラグ
}
```

#### 1. 語彙レベルチェック

- [ ] 中級に上級語彙が含まれていないか

## 🔧 作成ワークフロー

### Phase 1: 準備
1. テーマに関連する語彙リストをCSVから抽出
1. 語彙カバレッジ計画を立てる
1. アウトラインを作成（段落構成）

### Phase 2: 本文作成
1. 英文を作成（目標語数を意識）
1. 語彙レベルを確認しながら調整
1. 自然な流れと文法正確性を確保

### Phase 3: チャンク分割
1. 意味の塊で文章を分割
1. 各チャンクの長さを確認
1. 自然な区切りかを音読で確認

### Phase 4: 直訳作成
1. 各単語の直訳を作成
1. 文法要素の直訳ルールを適用
1. フレーズ全体の和訳を作成

### Phase 5: 検証
1. 自動検証スクリプトで語彙レベルチェック
1. 語数カウント確認
1. 手動で内容と自然さを確認

### Phase 6: JSON化
1. データ構造に変換
1. フォーマット確認
1. passages.json に追加
