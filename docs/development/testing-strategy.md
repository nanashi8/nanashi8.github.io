# 24. テスト戦略仕様書

## 🧪 概要

本アプリケーションのテスト戦略と品質保証の方針を説明します。手動テスト中心のアプローチを取りつつ、将来的な自動テスト導入の方針も示します。

---

## 🎯 テスト方針

### 1. 現状のテストアプローチ

- **手動テスト中心**: 主要機能を手動で確認
- **ブラウザテスト**: Chrome, Safari, Firefoxで動作確認
- **レスポンシブテスト**: モバイル・タブレット・デスクトップ
- **型チェック**: TypeScriptによる静的型検証

### 2. テスト項目

#### 機能テスト

- [ ] 和訳クイズ: 選択肢の表示・正誤判定・音声再生
- [ ] スペルクイズ: 入力・ヒント・バリデーション
- [ ] 長文読解: 未知語マーク・段階的和訳・問題集生成
- [ ] 統計画面: グラフ表示・定着率計算・履歴
- [ ] 設定画面: AI人格変更・ダークモード切り替え

#### パフォーマンステスト

- [ ] 初回読み込み時間: 3秒以内
- [ ] データ読み込み: 1秒以内
- [ ] 画面遷移: 即座に反応

#### データ整合性テスト

- [ ] LocalStorage保存・読み込み
- [ ] IndexedDB保存・読み込み
- [ ] 進捗データの正確性

---

## 🔍 テスト実行方法

### 手動テスト手順

1. `npm run dev` で開発サーバー起動
1. ブラウザで各機能を操作
1. DevToolsでエラーログを確認
1. LocalStorageのデータを確認

### TypeScript型チェック

```bash
npx tsc --noEmit
```

### ビルドテスト

```bash
npm run build
npm run preview
```
