# コンテンツ品質原則 (Content Quality Principles)

**最終更新**: 2025年12月13日

## 基本方針

このプロジェクトは**質を最優先**します。

```
質 > スピード
質 > 量
質 > 効率
```

## 品質の定義

### 1. 分かりやすさ (Clarity)
- 学習者が理解しやすい表現
- 曖昧さのない明確な説明
- 具体例を含む

### 2. 教育的価値 (Educational Value)
- 正答だけでなく「なぜそうなるか」を説明
- 関連知識への拡張
- 学習の文脈を提供

### 3. 正確性 (Accuracy)
- 文法的に正確
- 用語の正しい使用
- 誤解を招かない表現

### 4. 完全性 (Completeness)
- 必要な情報がすべて含まれている
- 学習に必要な要素が揃っている
- 段階的な理解を促進

## explanation(解説)の品質基準

### 必須要素

1. **文法ルールの説明**
   - どの文法項目が使われているか
   - その文法ルールは何か

2. **正答の明示**
   - 正答が何か
   - なぜそれが正答か

3. **具体的な説明**
   - 一般論だけでなく、この問題固有の説明
   - 学習者が「なるほど」と思える内容

### 良い例

```json
{
  "correctAnswer": "spoken",
  "explanation": "受動態では過去分詞を使います。speakの過去分詞はspokenです。「英語は話されている」という受け身の意味になります。"
}
```

### 悪い例（修正が必要）

```json
{
  "correctAnswer": "spoken",
  "explanation": "受動態では過去分詞を使う。"
}
```
→ 正答が明示されていない、具体性に欠ける

## hint(ヒント)の品質基準

### 必須要素

1. **考えるヒント**
   - 直接答えを言わない
   - 思考の方向性を示す

2. **関連知識**
   - 既習事項との関連
   - 類似パターンの提示

3. **段階的支援**
   - 最初は軽いヒント
   - 必要に応じてより具体的に

## japanese(日本語訳)の品質基準

### 必須要素

1. **自然な日本語**
   - ネイティブスピーカーが使う表現
   - 文脈に合った訳

2. **英文との対応**
   - 意味が正確に対応
   - ニュアンスが保たれている

3. **学習者への配慮**
   - 難しすぎない表現
   - 必要に応じて補足

## 品質保証プロセス

### Phase 1: データ作成
- 品質基準に基づいて作成
- レビューチェックリスト使用

### Phase 2: 自動検証
- テストによる基本的な品質チェック
- 構造的な問題の検出

### Phase 3: 品質向上
- テスト結果に基づく改善
- **質を妥協しない**：時間がかかっても全問を修正

### Phase 4: 継続的改善
- 学習者フィードバックの収集
- 定期的な見直しと更新

## 時間 vs 質のトレードオフ

### 原則

**時間を犠牲にしても質を優先する**

- 169問の修正が必要なら、169問すべてを修正する
- テストを緩和して問題を隠さない
- 「とりあえず動く」ではなく「本当に良い」を目指す

### 理由

1. **学習効果**: 質の高いコンテンツは学習効果が何倍にもなる
2. **信頼性**: 学習者の信頼を得られる
3. **長期的価値**: 一度質を上げれば、その価値は長く続く
4. **技術的負債の回避**: 妥協したコンテンツは将来的に大きな問題になる

## レビュー基準

### 各問題をレビューする際の質問

1. **分かりやすいか？**
   - 中学生が読んで理解できるか
   - 専門用語は適切に説明されているか

2. **教育的か？**
   - 学習者の成長につながるか
   - 「なぜ」が説明されているか

3. **正確か？**
   - 文法的に正しいか
   - 誤解を招く表現はないか

4. **完全か？**
   - 必要な情報はすべて含まれているか
   - 関連知識への言及があるか

## 実装方針

### ツールとプロセス

1. **自動検証**: 基本的な品質チェック（構造、必須フィールド等）
2. **品質スクリプト**: explanationの質を自動向上
3. **手動レビュー**: 最終的な品質確認
4. **継続的改善**: フィードバックループの構築

### 優先順位

1. **正確性**: まず正しいこと
2. **完全性**: 次に必要な情報がすべて含まれること
3. **分かりやすさ**: そして理解しやすいこと
4. **教育的価値**: 最後に学習効果を最大化すること

---

## まとめ

> 「質の高い10問は、質の低い100問に勝る」

このプロジェクトでは、量やスピードよりも**質**を最優先します。
時間がかかっても、学習者にとって最高の教育体験を提供することを目指します。
