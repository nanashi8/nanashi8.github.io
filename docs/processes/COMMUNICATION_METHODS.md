# コミュニケーション手法

## 概要

この文書は、開発者とAIアシスタントが作業を進める際のコミュニケーション（依頼→確認→合意→実装→検証）の最小手順をまとめる。

---

## 最小プロトコル（提案）

1. **目的**：何を達成したいか（1文）
2. **制約**：触ってはいけない境界、禁止事項
3. **受け入れ条件**：完了の定義（テスト/ログ/見た目など）
4. **確認質問**：曖昧さがある場合は先に質問して確定
5. **差分最小**：変更範囲を狭く保つ
6. **検証**：該当する品質ゲート/テストを実行
