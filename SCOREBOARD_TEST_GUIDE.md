# スコアボードテストガイド

## テスト方法

### 1. 開発環境でのテスト

開発サーバーを起動後、ブラウザのコンソール（F12）を開いて以下のコマンドを実行してください：

```javascript
// 全てのテストを実行
window.runScoreBoardTests();
```

### 2. 手動テスト項目

#### デスクトップ版
- [ ] 4つのタブ（今日、週間、月間、全期間）が正しく表示される
- [ ] 各タブをクリックして切り替えできる
- [ ] 各タブで正しい統計データが表示される
- [ ] 正答率が0-100%の範囲で表示される
- [ ] 連続正解数が整数で表示される
- [ ] レイアウトが崩れていない

#### モバイル版（画面幅768px未満）
- [ ] タブナビゲーション（◀ 今日 ▶）が表示される
- [ ] 左ボタン（◀）をクリックして前のタブに移動できる
- [ ] 右ボタン（▶）をクリックして次のタブに移動できる
- [ ] 最初のタブで左ボタンを押すと最後のタブに移動する（ループ）
- [ ] 最後のタブで右ボタンを押すと最初のタブに移動する（ループ）
- [ ] 統計データが正しく表示される
- [ ] 縦方向のスペースが節約されている

### 3. レスポンシブデザインのテスト

ブラウザの開発者ツールで以下の画面サイズをテスト：
- [ ] 320px（小型スマートフォン）
- [ ] 375px（iPhone SE）
- [ ] 768px（タブレット）
- [ ] 1024px（デスクトップ）
- [ ] 1920px（大型デスクトップ）

### 4. データ整合性のテスト

以下のシナリオで統計が正しく更新されるか確認：
- [ ] 問題に正解した後、正答率が上がる
- [ ] 問題に不正解した後、正答率が下がる
- [ ] 連続で正解すると、連続正解数が増える
- [ ] 不正解後、連続正解数がリセットされる
- [ ] 日付が変わった後、「今日」の統計がリセットされる

### 5. エッジケースのテスト

- [ ] データがない状態（初回起動時）で正しく表示される
- [ ] 極端に多い問題数（1000問以上）でパフォーマンスが保たれる
- [ ] 100%正答率の表示が正しい
- [ ] 0%正答率の表示が正しい
- [ ] 連続正解数が0の時の表示が正しい

## 既知の問題

現在のところ、既知の問題はありません。問題を発見した場合は、以下の情報と共に報告してください：
- 発生した問題の詳細
- 再現手順
- 使用しているブラウザとバージョン
- 画面サイズ
- コンソールのエラーメッセージ（あれば）
