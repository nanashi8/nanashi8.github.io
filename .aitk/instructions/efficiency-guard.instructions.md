# ⚡ 時間効率ガード

## 🎯 目的

「時間がかかりすぎています」という指摘があった作業を記録し、同じ無駄な作業を自動的に防止します。

## 🚫 禁止作業リスト

### 1. 個人開発でのPR作成

❌ **やらない**: GitHub PRの作成・レビュー待ち  
✅ **やる**: `git merge` → `git push` で直接マージ

### 2. 過度な説明・サマリー

❌ **やらない**: 長文の完了レポート、詳細なサマリー  
✅ **やる**: 3行以内の簡潔な報告

### 3. 不要なドキュメント作成

❌ **やらない**: ユーザーが要求していないドキュメント  
✅ **やる**: コード内コメントのみ、または明示的に依頼された場合のみ

### 4. 冗長な確認作業

❌ **やらない**: 実装後の長い動作確認レポート  
✅ **やる**: テストが通ればOK、簡潔に報告

## 📋 標準作業フロー

```
実装 → テスト → マージ → 「完了」(1行)
```

## 🛡️ サーバント連携

以下のフレーズを検出した場合、即座にガードを発動：

- "時間がかかりすぎています"
- "時間がかかりすぎている"
- "遅すぎます"
- "早くしてください"

### ガード発動時の対応

1. **即座に中断**: 現在の作業を停止
2. **最小限の完了**: 必要最小限の作業のみ実行
3. **簡潔な報告**: 1-3行で結果を報告

## 🔧 実装ガイド

### AI Assistantへの指示

```
- PRは作成しない（個人開発）
- ドキュメントは最小限
- 説明は3行以内
- 作業完了は「✅ 完了」のみ
```
