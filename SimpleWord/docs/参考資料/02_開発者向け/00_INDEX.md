## 📚 ドキュメント一覧

### 1. ファイル分割ガイド.md 🔧 コード整理の基本
**推奨学習時間**: 20分

**内容**:
- ファイル分割のタイミング
- 分割の具体的な方法
- Feature-First における分割パターン
- ベストプラクティス

**こんな時に読む**:
- ファイルが大きくなってきた（200行以上）
- 複数の責務が混在している
- テストが書きにくくなった
- コードの見通しが悪くなった

**実例**:
- View の部品化
- ViewModel のロジック分離
- Services への責務移動

## 🎯 利用シーン別ガイド

### シーン1: ファイルが大きくなってきた
1. **ファイル分割ガイド.md** を読む
2. 分割タイミングを確認（200〜300行）
3. 責務を分析
4. 段階的に分割

### シーン2: 機能が動かなくなった
1. **機能復元ガイド.md** を読む
2. 優先度を確認（P0〜P3）
3. 参照ドキュメントを確認
4. 段階的に復元

### シーン3: 大規模なリファクタリングをしたい
1. **リファクタリング実践ガイド.md** を読む
2. 準備（バックアップ、ブランチ作成）
3. Phase 1: Models/Services 分離
4. Phase 2: UI コンポーネント分離
5. Phase 3: 最終調整

### シーン4: 適応型学習機能を理解したい
1. **ADAPTIVE_LEARNING_GUIDE.md** を読む
2. MemoryStageの実装を確認
3. QuizViewの統合部分を確認
4. 必要に応じてカスタマイズ

### シーン5: コード品質を向上させたい
1. **DEPRECATION_GUIDE.md** を読む
2. SwiftLintをインストール
3. pre-commitフックを設定
4. 定期的にチェックを実行

### シーン6: テスト・動作確認をしたい
1. **TEST_GUIDE.md** を読む
2. ビルド＆実行
3. CSV読み込みテスト
4. パフォーマンス確認
