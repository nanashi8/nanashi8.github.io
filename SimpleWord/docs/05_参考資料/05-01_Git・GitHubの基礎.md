## Gitって何？

### 🎮 ゲームのセーブデータに例えると...

**Git** は、プログラムの「セーブ機能」です。

- **セーブポイント（コミット）**: ゲームの「セーブデータ」と同じ。いつでも戻れる
- **セーブファイル名（コミットメッセージ）**: 「ボスを倒した後」「レベル50になった時」みたいなメモ
- **複数のセーブデータ（ブランチ）**: 「攻略ルートA」「攻略ルートB」を別々に試せる
- **オンラインストレージ（GitHub）**: セーブデータをクラウドに保存。パソコンが壊れても安心

## 用語解説（辞書）

### 基本用語

| 用語 | 読み方 | 意味 | 例え |
|------|--------|------|------|
| **リポジトリ (Repository)** | リポジトリ | プロジェクト全体の保管場所 | ゲームのセーブデータフォルダ |
| **コミット (Commit)** | コミット | 変更を記録すること | ゲームをセーブする |
| **ブランチ (Branch)** | ブランチ | 作業の枝分かれ | 別の攻略ルートを試す |
| **マージ (Merge)** | マージ | ブランチを統合すること | 別ルートの成果を本編に合体 |
| **プッシュ (Push)** | プッシュ | ローカルの変更をリモートへ送る | セーブをクラウドにアップロード |
| **プル (Pull)** | プル | リモートの変更をローカルへ取得 | クラウドから最新版をダウンロード |
| **クローン (Clone)** | クローン | リモートのコピーを作る | プロジェクトを初めてダウンロード |

### ローカル vs リモート

| 用語 | 意味 |
|------|------|
| **ローカル (Local)** | あなたのパソコンの中 |
| **リモート (Remote)** | インターネット上（GitHub）|

**イメージ**:
```
ローカル（自分のPC）          リモート（GitHub）
    📁                  ←push→      ☁️
    自分だけ見える                みんなで共有
```

### ブランチの種類

| ブランチ名 | 役割 |
|-----------|------|
| **main** | 正式版。完成したコードが入る（絶対に壊してはいけない） |
| **develop** | 開発版。新機能を統合する場所 |
| **feature/〇〇** | 機能追加用。自分の作業場所 |
| **fix/〇〇** | バグ修正用 |

**イメージ**:
```
main       ─────●─────●─────●  正式版（安全）
             ↗      ↗      ↗
develop    ──●──●──●──●──●──●  開発版（テスト中）
             ↗  ↗     ↗  ↗
feature/A  ●──●         　      新機能Aを作成中
feature/B       ●──●──●          新機能Bを作成中
```

## Gitの仕組み（イメージ）

### 📦 変更が記録される流れ

```
1. ファイルを編集する（作業）
   📝 コードを書く
   
2. ステージに追加する（git add）
   📋 「これをセーブに含める」と指定
   
3. コミットする（git commit）
   💾 セーブ完了！メッセージを付ける
   
4. プッシュする（git push）
   ☁️ クラウド（GitHub）にアップロード
```

### 🌳 ブランチのイメージ

```
main ブランチ（正式版）
  │
  ├─── feature/new-function ブランチ（新機能開発）
  │       │
  │       ├─ ファイルAを編集
  │       ├─ ファイルBを追加
  │       └─ コミット「新機能完成」
  │
  └─── 後でマージ（合体）→ main に反映
```

## よく使うコマンド一覧

### 🔰 超基本（これだけ覚えればOK）

| コマンド | 意味 | 使うタイミング |
|---------|------|---------------|
| `git status` | 今の状態を確認 | 困ったらまずこれ |
| `git add .` | 全部の変更をステージに追加 | コミットする前 |
| `git commit -m "メッセージ"` | セーブする | 変更をまとめる時 |
| `git push` | リモートに送る | 作業を共有する時 |
| `git pull` | 最新版を取得 | 作業を始める前 |

### 📊 状態確認系

```bash
# 今の状態を見る
git status

# 変更内容を見る
git diff

# コミット履歴を見る
git log --oneline --graph
```

### 🌿 ブランチ操作系

```bash
# ブランチ一覧を見る
git branch

# 新しいブランチを作る
git branch feature/my-work

# ブランチを切り替える
git switch feature/my-work

# 作成と切り替えを同時に行う
git switch -c feature/my-work

# ブランチを削除する
git branch -d feature/my-work
```

### 💾 保存系

```bash
# ファイルをステージに追加
git add ファイル名
git add .                    # 全部

# コミット（セーブ）する
git commit -m "変更内容のメモ"

# リモートに送る（アップロード）
git push origin ブランチ名
```

### 🔄 同期系

```bash
# リモートの最新情報を取得（まだ合体しない）
git fetch origin

# リモートの最新版を取得して合体
git pull origin main

# 他のブランチを今のブランチに合体
git merge feature/my-work
```

## 🎯 実際の作業例

### 例1: 新機能を作る

```bash
# 1. 最新版を取得
git pull origin main

# 2. 新しいブランチを作って移動
git switch -c feature/add-quiz-timer

# 3. コードを編集（エディタで作業）

# 4. 状態を確認
git status

# 5. 変更をステージに追加
git add .

# 6. コミット（セーブ）
git commit -m "Add quiz timer feature"

# 7. リモートにプッシュ
git push origin feature/add-quiz-timer
```

### 例2: バグを修正する

```bash
# 1. 修正用ブランチを作成
git switch -c fix/quiz-crash-bug

# 2. バグを修正（エディタで作業）

# 3. 変更をコミット
git add .
git commit -m "Fix quiz crash when no data"

# 4. プッシュ
git push origin fix/quiz-crash-bug
```

## 🆘 困った時のチェックリスト

### Q1: 今どこにいるの？
```bash
# 現在のブランチを確認
git branch
# * マークが付いているのが今いる場所
```

### Q2: 何を変更したっけ？
```bash
# 変更したファイルを見る
git status

# 詳しい変更内容を見る
git diff
```

### Q3: コミットしたけど間違えた！
```bash
# 直前のコミットを取り消す（変更は残る）
git reset --soft HEAD~1

# やり直してコミット
git add .
git commit -m "正しいメッセージ"
```

### Q4: ブランチを間違えた！
```bash
# 正しいブランチに移動
git switch 正しいブランチ名

# さっきの変更を持ってくる（必要なら）
git cherry-pick コミットID
```

## 💡 覚えておくと便利なこと

### エイリアス（ショートカット）設定

よく使うコマンドを短くできます：

```bash
# 設定
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.cm commit

# 使い方
git st        # git status の省略形
git co main   # git checkout main の省略形
```

### VS Codeと連携

VS Codeには Git が組み込まれています：

- **ソース管理パネル**: 左サイドバーのブランチアイコン
- **変更の確認**: 黄色や緑の線でわかる
- **コミット**: メッセージを入れて「✓ コミット」ボタン
- **プッシュ**: 「...」→「プッシュ」
