# Xcode・Swift問題CSV作成・拡充 Instruction

この文書は、プロジェクトで使用する `xcode.csv` を一貫して、かつ手作業で作成・拡充しやすくするための具体的な作成要領です。

## 目的
- Xcode開発で頻出する英単語、コードパターン、構文を「単語」「熟語」のように出題できる形式で整備する
- SwiftやXcodeの学習において、コードの意味と英語の語源を理解することで記憶定着率を向上させる
- プログラミング初心者が効率的にXcode/Swift用語を習得できるようにする

## 前提
- 文字コード: UTF-8（BOM なし）
- 区切り文字: カンマ ( , )
- 改行: LF 推奨（macOS 標準）
- ファイル先頭に必ずヘッダ行を残す（下記ヘッダ順を厳守）

## 必須ヘッダ（順序固定）
1. 語句
2. 発音（カタカナ）
3. 和訳
4. 語源等解説（日本語）
5. 関連語（英語）と意味（日本語）
6. 関連分野（日本語）
7. 難易度

## 各列の定義と入力ルール

### 語句
- **英語の見出し語またはコードパターン**
- **3つのタイプ**:
  1. **単一単語**: `variable`, `function`, `class`
  2. **複合語**: `viewController`, `addSubview`
  3. **コード構文**: `func name()`, `if condition`, `let constant`
- 小文字で記載（定数は大文字可: `NSNotification`）
- キャメルケースはそのまま: `viewDidLoad`, `UIViewController`
- コード構文の場合、型や変数名は汎用的なものを使用

**良い例**:
```
variable
func
viewDidLoad
UIButton
if condition
for item in array
```

**避けるべき例**:
```
myVariable  // 具体的すぎる
func mySpecificFunction()  // 長すぎる
```

### 発音（カタカナ）
- 日本語カタカナ表記で入力
- プログラミング用語の慣用的な読み方を記載
- 略語の場合はアルファベット読み: `UI` → `ユーアイ`
- キャメルケースは区切りを意識: `viewDidLoad` → `ビュー・ディド・ロード`

**例**:
```
variable → ヴァリアブル
func → ファンク
UIViewController → ユーアイ・ビュー・コントローラー
@IBOutlet → アット・アイビー・アウトレット
```

### 和訳
- プログラミング文脈での意味を記載
- 一般的な英単語の訳ではなく、コード上の役割を説明
- 簡潔に（20文字以内を目安）

**例**:
```
variable → 変数
func → 関数
viewDidLoad → ビューが読み込まれた時
override → 上書き・オーバーライド
```

### 語源等解説（日本語）
- **目的**: 英単語の語源と、プログラミング文脈での使われ方を理解する
- **文字数**: 40〜250 文字程度
- **必須ではないが、可能な限り埋める**

**記述内容（優先順）**:

1. **英単語の語源と本来の意味**（最優先）
   - 例: "variable: ラテン語variābilis（変わりやすい）←variāre（変える）。値が変化できる入れ物の意"
   - 例: "function: ラテン語functiō（遂行）←fungī（果たす）。特定の役割を果たすコードのまとまり"
   - 例: "override: over（上に）+ ride（乗る）。親クラスのメソッドの上に乗って書き換える"

2. **プログラミングでの役割・使い方**
   - 例: "let: 「させる」の意。値を定数に代入する。一度代入したら変更不可"
   - 例: "guard: 「守る」の意。条件を満たさない場合に早期リターンして安全性を守る"
   - 例: "weak: 「弱い」の意。循環参照を防ぐため弱い参照を作る"

3. **Swiftでの具体的な使用例**
   - 例: "viewDidLoad: ビューが読み込まれた直後に呼ばれる。初期設定はここで行う"
   - 例: "@IBOutlet: Interface BuilderのUI部品とコードを接続する修飾子"
   - 例: "optional: 値があるかないか（nil）を表現する型。?や!で扱う"

4. **同じ語源・パターンを持つ関連語（抱き合わせ記憶）**（重要）
   - 例: "view: ラテン語vidēre（見る）。同語源：viewController（表示を制御）、viewDidLoad（表示が読み込まれた）、subview（子ビュー）"
   - 例: "init: initialize（初期化）の略。同語源：deinit（解放）、initialized（初期化済み）"
   - 例: "set-: 「設定する」接頭辞。同パターン：setTitle（タイトル設定）、setImage（画像設定）、setBackgroundColor（背景色設定）"

5. **対になる概念・反対語**
   - 例: "strong: 強い参照（デフォルト）。対義語：weak（弱い参照）"
   - 例: "push: ナビゲーションで画面を積む。対義語：pop（画面を取り除く）"
   - 例: "encode: 符号化する。対義語：decode（復号化する）"

**記述の原則**:
- ✓ 英単語の語源（ラテン語、ギリシャ語等）を明記する
- ✓ プログラミングでの意味と一般的な意味の関連を説明する
- ✓ Swiftでの具体的な使用場面を示す
- ✓ 同じ語源・パターンの関連語を「同語源：」「同パターン：」で明記
- ✓ コード例は汎用的で短いものにする
- ✓ 初心者にもわかりやすい説明を心がける

**避けるべき説明**:
- ✗ 語源のない表面的な説明（例: "変数です"だけ）
- ✗ 専門的すぎる詳細（例: "ARC のリテンションカウントが..."）
- ✗ 特定のプロジェクト固有の例
- ✗ 長すぎる説明（250文字超）

**良い例**:
```
delegate: ラテン語dēlegāre（委任する）←dē（下へ）+ legāre（送る）。処理を委任するデザインパターン。同語源：delegation（委任）

optional: option（選択肢）+ -al（形容詞）。値があるかnil（なし）かを選択できる型。?で宣言、!で強制展開

addSubview: add（追加）+ sub（下）+ view（ビュー）。親ビューの下に子ビューを追加。同パターン：removeFromSuperview（親から削除）
```

### 関連語（英語）と意味（日本語）
- フォーマット: `英語:日本語` のペアをセミコロン (;) で区切る
- 関連する単語・構文を3〜5個程度挙げる
- 対義語、派生語、セットで使う語を優先

**例**:
```
variable → var:変数宣言; let:定数宣言; constant:定数
func → function:関数; method:メソッド; return:返り値
UIButton → UILabel:ラベル; UIImageView:画像ビュー; @IBAction:アクション接続
```

### 関連分野（日本語）
以下の20カテゴリから**必ず1つ**を選択:

1. Swift基礎
2. UI構築
3. ライフサイクル
4. データ型
5. 制御構文
6. 関数・メソッド
7. クラス・構造体
8. プロトコル
9. エラー処理
10. メモリ管理
11. 非同期処理
12. データ永続化
13. ネットワーク
14. デバッグ
15. テスト
16. アニメーション
17. センサー・位置情報
18. 通知
19. アーキテクチャ
20. その他

**カテゴリの選び方**:
- 基本的な単語: `Swift基礎`（var、let、if、for等）
- UI関連: `UI構築`（UIButton、UILabel、addSubview等）
- ビューの表示・非表示: `ライフサイクル`（viewDidLoad、viewWillAppear等）
- 型: `データ型`（String、Int、Array、Optional等）
- if/for/while等: `制御構文`
- メソッド関連: `関数・メソッド`
- class/struct: `クラス・構造体`

### 難易度
- 1〜3 の整数
- **1**: 初心者レベル（変数、関数、基本的なUI部品）
- **2**: 中級者レベル（delegate、optional、ライフサイクル）
- **3**: 上級者レベル（メモリ管理、プロトコル、非同期処理）

## フォーマット上の技術的注意点
- カンマ区切りのため、各セル内でカンマを使用しないこと
- コード例で必要な場合は全角カンマ「，」を使用
- セルに改行を含めない
- コード構文の場合、説明に必要な最小限の形にする

## 入力チェックリスト
- [ ] ファイルが UTF-8（BOM なし）で保存されている
- [ ] ヘッダが正しい順序で1行目にある
- [ ] 各行の列数が7列になっている
- [ ] `語句` が適切な形式（単語、複合語、コード構文）
- [ ] `関連語` のペアは `英語:日本語` で、複数は `;` で区切り
- [ ] `関連分野` が20カテゴリのいずれか1つ
- [ ] `難易度` が 1〜3 のいずれか

## 出題内容の方針

### 優先的に含めるべき内容
1. **頻出単語**: var, let, func, class, struct, if, for, return
2. **UI部品**: UIButton, UILabel, UIImageView, UITableView
3. **重要な修飾子**: @IBOutlet, @IBAction, override, weak, strong
4. **ライフサイクル**: viewDidLoad, viewWillAppear, viewDidAppear
5. **Swift特有の概念**: optional, guard, defer, extension
6. **よく使うメソッド**: addSubview, addTarget, reloadData
7. **データ型**: String, Int, Double, Bool, Array, Dictionary
8. **エラー処理**: try, catch, throw, Error
9. **プロトコル**: delegate, protocol, Codable, Equatable

### 段階的な学習順序
- **レベル1（基礎）**: 変数、データ型、制御構文、基本UI部品
- **レベル2（実践）**: ライフサイクル、delegate、optional、メソッド
- **レベル3（応用）**: メモリ管理、プロトコル、非同期、アーキテクチャ

## よくあるミスと対処法
- **コード構文が長すぎる**: 最小限の形に簡略化する
- **専門用語だけで説明**: 英語の語源も含めて説明する
- **関連分野の誤選択**: カテゴリ定義を確認する
- **カンマをセル内で使用**: 全角カンマ「，」に置換
