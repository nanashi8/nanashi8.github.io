**最速で問題解決するための簡易ガイド**

## 🚀 3秒でわかるコツ

### ❌ ダメな例
「バグがあります」

### ✅ 良い例
「QuizView で選択肢の順序が CSV と違います。
CSV: A→B→C→D
UI: A→C→D→B」

---

## 📝 状況別テンプレート

### バグ報告
```
【症状】[何が起きているか]
【期待】[本来どうあるべきか]
【差分】[何が違うか]
【ファイル】[該当ファイル（推測でも可）]
```

**例**:
```
【症状】選択肢の表示順: 年号→史実名→解説→登場人物
【期待】CSV の列順: 年号→登場人物→史実名→解説
【差分】2番目と3番目が入れ替わっている
【ファイル】ChoiceCardView.swift
```

---

### 機能変更
```
【現在】[今の動作]
【変更後】[こうしたい]
【理由】[なぜ変更したいか]
```

**例**:
```
【現在】正解時のみ1.5秒後に自動進行
【変更後】正解・不正解問わず2秒後に自動進行
【理由】不正解時もテンポよく進みたい
```

---

### 機能削除
```
【削除対象】[何を削除したいか]
【理由】[なぜ不要か]
【代替】[代わりにどうするか]
```

**例**:
```
【削除対象】NavigatorView（単語リスト機能）
【理由】使用されておらず、保守コストがかかる
【代替】学習結果画面から単語確認可能
```

---

### 新機能追加
```
【機能名】[何を作りたいか]
【目的】[なぜ必要か]
【仕様】
  入力: [ユーザーの操作]
  処理: [内部処理]
  出力: [結果]
```

**例**:
```
【機能名】学習履歴のエクスポート
【目的】バックアップと分析のため
【仕様】
  入力: 学習結果画面の「エクスポート」ボタン
  処理: ScoreStore を CSV 化
  出力: 共有シートで CSV ファイル
```

## 💡 効率化の鉄則

### 1. 具体的な数値を使う
❌ 「遅い」
✅ 「5秒かかる（期待: 1秒以内）」

### 2. ファイル名を明記
❌ 「表示がおかしい」
✅ 「ChoiceCardView.swift の表示がおかしい」

### 3. 期待と実際を並べる
```
期待: A → B → C
実際: A → C → B
```

### 4. スクリーンショットも有効
画像1枚で1000語の説明に匹敵

---

## 🎯 情報の優先順位

### 最優先（必須）
- **何が問題か**（症状）
- **どうあるべきか**（期待）

### 高優先（あると良い）
- 該当ファイル名
- 再現手順
- 差分の明確化

### 中優先（さらに良い）
- 環境情報（デバイス、iOS バージョン）
- エラーメッセージの全文
- 関連する他の動作

### 低優先（任意）
- 推測される原因
- 修正案
- 優先度

## ⚡ よくある質問

### Q: 原因がわからなくても大丈夫？
A: はい！「何が起きているか」と「どうあるべきか」を伝えれば OK

### Q: 専門用語がわからない
A: 日常の言葉で OK
- 「画面」= View
- 「データ」= Model
- 「処理」= ViewModel / Logic

### Q: 長文になってしまう
A: 箇条書きで簡潔に！段落分けも効果的
