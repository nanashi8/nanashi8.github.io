# CSS修正ルール

## 重要原則

### 🚫 禁止事項
1. **ユーザーの明示的な指示なしに、以下を変更してはいけません:**
   - レイアウト構造 (flex, grid, position等)
   - デザイン要素 (border-radius, padding, margin, gap等)
   - フォントサイズ
   - アニメーション
   - 要素の配置

1. **ライトモードへの影響を与えない:**
   - ダークモード修正時は必ず `.dark-mode` クラス内で行う
   - ライトモードのスタイルは変更しない

### ✅ 許可される修正

1. **ダークモード専用の色変更 (ユーザー指示時のみ):**
   ```css
   /* ✅ OK: ダークモード専用 */
   .dark-mode .element {
     background: var(--gray-900);
     color: var(--gray-100);
   }
   
   /* ❌ NG: ライトモードに影響 */
   .element {
     background: var(--gray-900);
   }
   ```

1. **バグ修正:**
   - TypeScriptエラー
   - 構文エラー
   - 機能不全の修正

## ダークモード色ガイドライン

### 背景色
- **主背景:** `var(--gray-950)` または `var(--gray-900)` (黒っぽく)
- **副背景:** `var(--gray-900)` または `var(--gray-800)`
- **カード背景:** `var(--gray-900)` または `var(--gray-800)`

### 文字色
- **主文字:** `var(--gray-100)` または `var(--white)` (明るく)
- **副文字:** `var(--gray-200)` または `var(--gray-300)`
- **弱調文字:** `var(--gray-400)`

### ボーダー
- **通常:** `var(--gray-700)`
- **強調:** `var(--gray-600)`

## 修正前チェックリスト

- [ ] ユーザーから明示的な指示があるか？
- [ ] `.dark-mode` クラス内で修正しているか？
- [ ] ライトモードのスタイルに触れていないか？
- [ ] レイアウト・デザインを変更していないか？
- [ ] 色のみの変更か？

## 違反時の対応

指示なく修正した場合:
1. 即座に元に戻す
1. ユーザーに確認を求める
1. 明示的な許可を得てから実行

## 例外

以下の場合のみ、ユーザー指示なしで修正可能:
- TypeScriptコンパイルエラー
- 文法エラー
- 機能が完全に動作しない場合
